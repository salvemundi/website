Technisch Ontwerp: Datamodel & ERD Migratie Scope (Legacy -> V7)

Dit document bevat de genormaliseerde datastructuur binnen PostgreSQL (Directus) zoals deze in V7 komt te staan na de datamigratie. 
Op verzoek is hier de exacte lijst van alle 68 Legacy Postgres tabellen uitgedeeld in hun migratie-categorie, zodat we zeker weten dat er niets wordt vergeten.

---

### 1. Systeem & Identiteit (Directus -> Azure Entra ID Sync)
Dit is de kern van de integratie; Azure is de "Source of Truth".

*Tabellen die worden gemigreerd (aangepast of nieuw):*
- `directus_users`: Bestaande accounts worden gemerged o.b.v. `external_identifier` (Entra ID OID).
    - `date_of_birth` (Nieuw type in V7: `DATE`): In de legacy database was dit een `timestamp` (datetime). Dit wordt gemigreerd en gecast naar een zuivere `DATE`. Ontvangt 'YYYYMMDD' (bijv. 20060902) vanuit Azure CSA.
    - `lidmaatschap_verloopdatum` (Type: `DATE`): De exacte datum waarop het lidmaatschap verloopt. Ontvangt 'YYYYMMDD' string vanuit de Azure 'SalveMundiLidmaatschap' set `VerloopdatumStr`.
    - `originele_betaaldatum` (NIEUW, Type: `DATE`): De allereerste instroomdatum van een lid. Gekoppeld aan de CSA `OrigineleBetaalDatumStr`.
- `membership_history` (NIEUW): Voor tracking van lidmaatschapswijzigingen en statussen door de Sync Service.
- `azure_group_to_role` (Nieuw systeemconcept t.b.v. Azure sync)
